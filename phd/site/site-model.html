持续 Enrich 的 HTML

Continuously Enriched HTML

`$phd` 的长期输出不是一篇单次报告，而是一组可以持续生长的静态 HTML 页面。

The durable output of `$phd` is not a single report. It is a set of static HTML pages that can keep growing.

log.html带日期的 enrichment 记录和下一步目标。

Enrichment 规则

  • 修改前先读现有网站。
  • 保留有价值的既有工作，除非已经被取代。
  • 标记新增和实质变化内容。
  • 每次研究 pass 都增加日期日志。
  • 遇到重要概念、术语、缩写或专名时，更新 glossary.html。
  • 把建议的下一步写入 next-steps.html，等待用户确认。
  • 不要默认创建 actor-map.md、claims.md、todos.md 等 Markdown 研究产物。
  • 保持页面间导航链接可用。
  • 保持 file:// 友好，尽量少依赖。

为什么用 HTML

HTML 易导航、易检查、易链接，适合表格和结构化视觉表达，也可以从单页自然生长成不需要服务器的小网站。

log.htmlDated enrichment entries and next targets.

Enrichment Rules

  • Read the existing site before changing it.
  • Preserve useful prior work unless superseded.
  • Mark new and materially changed content.
  • Add a dated log entry for each research pass.
  • Update glossary.html when important concepts, terms, acronyms, or named entities appear.
  • Put proposed next actions in next-steps.html for user confirmation.
  • Do not create Markdown research artifacts such as actor-map.md, claims.md, or todos.md by default.
  • Keep navigation links working across pages.
  • Keep the site file:// friendly and dependency-light.

Why HTML

HTML is navigable, inspectable, easy to link, friendly to tables and visual structures, and can grow from one page into a small website without a server.
