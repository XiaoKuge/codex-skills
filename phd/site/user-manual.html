How To Use PhD Skill

`$phd` 是一个可组合的研究 skill，用于建立认知框架、survey 相关工作、综合复杂材料、提出假设，并持续 enrich 静态 HTML 研究网站。

`$phd` is a composable research skill for building frameworks, surveying related work, synthesizing complex material, generating hypotheses, and continuously enriching static HTML research sites.

简短版：不要只要求“回答”，而要要求“研究产出”。告诉 `$phd` 主题、资料或范围、深度、输出形态，以及是否需要更新 living HTML site。

什么时候使用

理解课题

把一个模糊课题转成清晰框架：定义、组成部分、关系、边界和开放问题。

调研相关工作

比较论文、项目、公司、人物、组织、学派或技术路线。

综合复杂材料

对复杂材料进行总结、抽象、组织和扩展，并从多个维度梳理。

生长研究网站

创建或 enrich 静态 HTML 网站，记录背景、相关工作、框架、假设、来源和日志。

如何写 Prompt

部分 说明什么 例子
主题说明概念、领域、问题、语料或项目。"research agents for literature review"
目标指定背景整理、survey、框架、综合、假设或网站更新。"build a concise cognitive framework and identify gaps"
范围限定资料、时间、人物组织、技术范围或篇幅。"focus on 2023-2026 systems and open-source projects"
输出指定产物形态，或让 `$phd` 判断。"produce a related work matrix and update the site"
持续性说明是否创建或更新 HTML research site。"enrich the existing site, add a log entry, and put proposed follow-ups in next-steps.html"

Prompt 模板

快速框架

使用 $phd 分析 [主题]。形成简洁认知框架：定义、组成、关系、边界、张力、相关概念、insights、hypotheses 和 open questions。

相关工作

使用 $phd survey [主题] 的相关工作。按 claim、方法、证据、假设、优点、局限和与我项目的关系进行比较。

复杂材料

使用 $phd 总结、抽象并扩展这些材料。从时间、因果、模块、行动者和失败模式等 lens 梳理。

持续网站

使用 $phd enrich 这个已有 research site。先读现有页面，保留已有工作，加入新发现，更新 glossary 和 hypotheses，并把建议的后续工作写入 next-steps.html 等我确认。

例子

例子 1：认知框架

用户：使用 $phd 分析 “agentic research systems”。我想快速获得完整 mental model。

预期：一页框架，包含定义、组成、流程、行动者、技术路线、失败模式、insights、hypotheses 和开放问题。

例子 2：相关工作 Survey

用户：使用 $phd 比较 AI agents for academic literature review 的相关工作。关注系统，不要泛泛比较聊天机器人。

预期：一个矩阵，按 workflow automation、search/retrieval、summarization、citation management、hypothesis generation 分组。

例子 3：复杂信息

用户：使用 $phd 处理这些笔记。总结内容，抽象机制，并按时间、因果、组织和模块梳理。

预期：一个综合报告，包含 timeline、causal model、actor map、module map、非显然 insights，以及清晰标注的不确定 claim。

例子 4：Enrich 研究网站

用户：使用 $phd 更新这个关于 “Codex automations” 的研究网站。增加 background、related work、glossary、hypotheses 和 log entry。把后续建议放到 next-steps.html，不要创建 todos.md。

预期：更新 HTML 页面，标记变化，增加 sources，补充 glossary，修订 hypotheses，并在 next-steps.html 中列出需要用户确认的下一步。

给 Agents

当 agent 调用 `$phd` 时，应有意识地选择 operators、lenses 和 output patterns。不要默认加载全部 reference，也不要默认生成所有 artifact。

  • operators.md：研究动作。
  • lenses.md：分析维度。
  • output-patterns.md：产物形态。
  • evidence-and-uncertainty.md：可靠性。
  • living-research-site.md：持续 HTML 网站工作。
Short version: Ask for a research outcome, not just an answer. Tell `$phd` the topic, sources or scope, desired depth, output shape, and whether it should update a living HTML site.

When To Use It

Understand a Topic

Turn a vague topic into a concise framework with definitions, components, relationships, boundaries, and open questions.

Survey Related Work

Compare papers, projects, companies, people, organizations, schools of thought, or technical routes.

Synthesize Material

Summarize, abstract, organize, and extend dense information across multiple dimensions.

Grow a Research Site

Create or enrich a static HTML site that records background, related work, frameworks, hypotheses, sources, and logs.

Prompt Templates

Fast Framework

Use $phd on [topic]. Build a concise conceptual framework: definition, components, relationships, boundaries, tensions, related concepts, insights, hypotheses, and open questions.

Related Work

Use $phd to survey related work on [topic]. Compare major works by claim, method, evidence, assumptions, strengths, limits, and relationship to my project.

Complex Material

Use $phd to summarize, abstract, and expand this material. Structure it across temporal, causal, modular, actor, and failure-mode lenses.

Living Site

Use $phd to enrich this existing research site. Read current pages first, preserve prior work, add findings, update glossary and hypotheses, and put proposed follow-ups in next-steps.html for my confirmation.

Examples

Example 1: Cognitive Framework

User: Use $phd to analyze "agentic research systems". I want a quick but complete mental model.

Expected: A one-page framework with definition, components, workflow, actors, technical routes, failure modes, insights, hypotheses, and open questions.

Example 2: Related Work Survey

User: Use $phd to compare related work on AI agents for academic literature review. Focus on systems, not generic chatbots.

Expected: A matrix grouped by workflow automation, search/retrieval, summarization, citation management, and hypothesis generation.

Example 3: Complex Information

User: Use $phd on these notes. Summarize them, abstract the mechanism, and organize by time, causality, organizations, and modules.

Expected: A synthesis with timeline, causal model, actor map, module map, non-obvious insights, and uncertain claims clearly marked.

Example 4: Enrich a Research Website

User: Use $phd to update this research site about "Codex automations". Add background, related work, glossary entries, hypotheses, and a log entry. Put proposed follow-ups in next-steps.html; do not create todos.md.

Expected: Updated HTML pages, marked changes, added glossary terms and sources, revised hypotheses, and proposed next steps waiting for user confirmation.

For Agents

When an agent invokes `$phd`, choose operators, lenses, and output patterns deliberately. Do not load every reference file or produce every artifact by default.

  • operators.md: research actions.
  • lenses.md: analytical dimensions.
  • output-patterns.md: artifact shapes.
  • evidence-and-uncertainty.md: reliability.
  • living-research-site.md: durable HTML site work.
