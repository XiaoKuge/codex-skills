# Research Operators

Use operators as composable actions. Pick only the operators needed for the current task.

## Background Map

Use when a topic lacks shared context.

Do:
- Define core terms.
- Identify adjacent concepts.
- Name key entities, systems, and stakeholders.
- List what must be understood before deeper analysis.
- Separate known background from assumptions.

Output:
- concise background brief
- term map
- prerequisite knowledge list
- key context bullets

## Related Work Survey

Use when comparing prior work, projects, papers, products, people, organizations, or schools of thought.

Do:
- Identify categories of work.
- Compare claims, methods, evidence, assumptions, and limitations.
- Map how works relate to each other.
- Name what each work explains well and what it misses.

Output:
- related work matrix
- cluster map
- evolution summary
- positioning memo

## Conceptual Framework

Use when a concept, topic, field, or vague direction needs a clear mental model.

Do:
- Define the topic in one sentence.
- Identify core components.
- Identify key variables and relationships.
- Clarify boundaries and non-examples.
- Surface tensions, tradeoffs, and failure modes.
- Name adjacent concepts.

Output:
- definition
- core components
- relationship map
- boundary conditions
- key questions
- minimal mental model

## Summarize

Use when compressing complex information without changing its conceptual level too much.

Do:
- Identify main claims and supporting details.
- Preserve important nuance.
- Remove repetition.
- Keep source-specific distinctions.

Output:
- executive summary
- structured notes
- claim list

## Abstract

Use when extracting higher-level patterns from detailed material.

Do:
- Identify mechanisms, invariants, recurring tensions, and hidden assumptions.
- Translate examples into generalizable concepts.
- Separate surface features from deeper structure.

Output:
- abstraction ladder
- pattern list
- mechanism description
- transferable model

## Expand

Use when extending implications from a concept or source material.

Do:
- Identify adjacent domains.
- Generate applications and analogies.
- Explore second-order effects.
- Name what would change under different constraints.

Output:
- implication map
- adjacent opportunities
- transfer notes
- future directions

## Compare

Use when two or more objects need structured contrast.

Do:
- Define comparison dimensions first.
- Compare like with like.
- Include assumptions, strengths, weaknesses, and contexts where each wins.

Output:
- comparison table
- decision matrix
- tradeoff memo

## Synthesize

Use when multiple sources or arguments must be integrated into one coherent view.

Do:
- Cluster related claims.
- Resolve compatible claims.
- Preserve unresolved contradictions.
- Create a higher-level framing that explains why differences exist.

Output:
- synthesis memo
- integrated framework
- disagreement map

## Trace Genealogy

Use when understanding historical development, intellectual lineage, or technical evolution.

Do:
- Identify phases.
- Name transition triggers.
- Track changing assumptions and capabilities.
- Link people, organizations, artifacts, and ideas.

Output:
- timeline
- phase table
- genealogy map

## Find Gaps

Use when looking for missing work, underexplored questions, or weak assumptions.

Do:
- Compare current claims with available evidence.
- Identify ignored cases, missing measurements, and brittle assumptions.
- Look for neglected intersections across lenses.

Output:
- gap list
- risk register
- open problem map

## Generate Insights

Use when the user asks for insight, framing, or non-obvious understanding.

Do:
- Look for surprising asymmetries, hidden constraints, category mistakes, incentives, and second-order effects.
- Tie each insight to observations or reasoning.
- Avoid generic observations.

Output:
- insight memo with evidence notes
- implications
- possible objections

## Generate Hypotheses

Use when the task needs testable explanations or research directions.

Do:
- State each hypothesis clearly.
- Explain why it might be true.
- Name what evidence would support it.
- Name what evidence would weaken or falsify it.

Output:
- hypothesis set
- test plan
- expected observations
- falsification criteria
